## What I do

I help create and manage static HTML website themes using Eleventy (11ty) as the static site generator and Tailwind CSS for styling. Themes are design templates used for client projects.

## When to use me

- Creating a new theme (e.g., plumber_theme, restaurant_theme)
- Editing existing theme structure
- Building or running theme dev server
- Understanding theme file organization

## Theme File Structure

```
theme-name/
├── .eleventy.js          # Eleventy configuration
├── package.json           # Dependencies & scripts
├── src/                  # INPUT - Source files (Eleventy reads from here)
│   ├── _includes/        # Reusable partials (header.html, footer.html)
│   ├── _layout/         # Base layouts (base.njk)
│   ├── css/             # CSS files
│   │   ├── base.css     # Fonts, sizing, spacing
│   │   ├── components.css # Buttons, inputs, UI components
│   │   ├── layout.css   # Header, hero, borders
│   │   ├── input.css    # Tailwind imports (what Tailwind watches)
│   │   └── output.css   # Compiled Tailwind output (generated)
│   ├── js/              # JavaScript files
│   │   ├── buttons.js
│   │   ├── header.js
│   │   └── [page-specific].js
│   ├── index.html       # Home page
│   ├── about.html      # About page
│   ├── [page].html     # Additional pages
├── _site/                # OUTPUT - Generated static site (don't edit)
├── node_modules/
└── guidelines.md
```

## Eleventy Configuration (.eleventy.js)

```js
module.exports = function(eleventyConfig) {
  eleventyConfig.addPassthroughCopy("src/css");
  eleventyConfig.addPassthroughCopy("src/js");

  return {
    dir: {
      input: "src",
      output: "_site",
      includes: "_includes",
      layouts: "_layout"
    }
  };
};
```

## Key Directories

| Directory | Purpose |
|-----------|---------|
| `src/` | Input folder - Eleventy reads templates from here |
| `_site/` | Output folder - Built static site goes here |
| `_includes/` | Reusable HTML partials (header, footer) |
| `_layout/` | Base Nunjucks templates that pages extend |
| `css/` | Tailwind source; output.css is generated |
| `js/` | JavaScript files |

## Components Structure

- **header.html** - Navigation header
- **footer.html** - Footer section
- **home_hero.html** - Hero section for homepage
- **page_hero.html** - Hero section for other pages
- **contact.html** - Contact section with form/CTA
- **button.html** - Button component preview

## Commands

```bash
# Start dev server (Eleventy + Tailwind watcher on port 8080)
npm start

# Build for production
npm run build

# Run Tailwind watcher only
npx tailwindcss -i ./src/css/input.css -o ./src/css/output.css --watch
```

## Styling

- Primary color: white (default), secondary: black
- Dark mode: swap primary/secondary
- Use Tailwind 4.0 or higher
- Only edit base.css, components.css, layout.css, input.css
- Never edit output.css - it's generated by Tailwind

## Naming Conventions

Theme names should follow pattern: `<name>_theme` (e.g., `plumber_theme`, `restaurant_theme`)
